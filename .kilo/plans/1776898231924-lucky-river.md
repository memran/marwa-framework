# Plan to Add Queue Worker Console Commands

## Issue Analysis
The user asks if console commands are available to execute queued tasks. The framework has queue functionality but lacks console commands to process queued jobs, preventing users from running queue workers.

## Root Cause
- Queue system exists but no console commands to execute jobs
- No `queue:work` or similar command to process pending jobs
- Users cannot run background workers to process email queues
- Missing infrastructure for queue processing

## Solution Overview
Add console commands for queue management and processing to enable users to execute queued email jobs.

## Implementation Steps
1. **Create Queue Work Command**: Add `queue:work` command to process jobs from queues
2. **Create Queue List Command**: Add `queue:list` command to show pending jobs
3. **Create Queue Clear Command**: Add `queue:clear` command to remove failed/stuck jobs
4. **Add Queue Configuration**: Ensure queue settings are configurable
5. **Implement Job Processing**: Handle job execution with error handling
6. **Add Worker Options**: Support queue selection, timeout, and retry logic
7. **Update Documentation**: Document queue worker usage
8. **Run Tests**: Ensure commands work correctly

## Files to Modify
- `src/Console/Commands/QueueWorkCommand.php`: New queue worker command
- `src/Console/Commands/QueueListCommand.php`: New queue list command
- `src/Console/Commands/QueueClearCommand.php`: New queue clear command
- `src/Config/QueueConfig.php`: Queue configuration (if needed)
- `docs/tutorials/queue.md`: Document queue commands

## Verification
- Test queue:work command processes jobs
- Test queue:list shows pending jobs
- Test queue:clear removes jobs
- Ensure email jobs are processed correctly
- Run full test suite
