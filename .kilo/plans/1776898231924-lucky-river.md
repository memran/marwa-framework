# Plan to Fix SwiftMailer Compatibility Issues in Marwa Framework

## Issue Analysis
The framework is encountering SwiftMailer compatibility errors due to version differences. The code uses deprecated static `newInstance()` methods that were removed in SwiftMailer 6.x, causing "Call to undefined method" errors.

## Root Cause
- SwiftMailer version mismatch: code expects 5.x API but may be running 6.x+
- `Swift_SmtpTransport::newInstance()` changed to `new Swift_SmtpTransport()`
- Similar issues with `Swift_SendmailTransport`, `Swift_MailTransport`, and `Swift_Message`
- `Swift_Attachment::fromPath()` and `newInstance()` also deprecated

## Solution Overview
Update all SwiftMailer instantiation calls to use the modern constructor syntax instead of deprecated `newInstance()` static methods.

## Implementation Steps
1. **Update Transport Instantiation**: Replace `::newInstance()` with `new` for all transport classes
2. **Update Message Creation**: Use `new Swift_Message()` instead of `Swift_Message()`
3. **Update Attachment Creation**: Use `new Swift_Attachment()` for attachments
4. **Check SwiftMailer Version**: Ensure compatibility with both old and new versions if needed
5. **Test All Mail Functions**: Verify SMTP, Sendmail, and Mail transports work
6. **Run Validation**: Execute tests and static analysis

## Files to Modify
- `src/Supports/Mailer.php`: Update all SwiftMailer instantiation calls

## Verification
- Test email sending with different transports
- Ensure no method undefined errors
- Run full test suite to prevent regressions

## Decision
Update to SwiftMailer 6.x+ modern constructor syntax, as the error indicates `newInstance()` is undefined. Add backward compatibility checks to support both old and new versions by detecting if `newInstance()` exists.
